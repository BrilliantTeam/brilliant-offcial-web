# 1. 請勿使用外掛 

包括 飛行、RNG計算器 ...等，
修改遊戲數據或嚴重影響遊戲公平性之行為。
違者 <span class="label label-danger">必定停權</span> 。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

# 2. 請勿冒犯他人

包括 威脅、淫穢、誹謗、仇恨、騷擾、侮辱、刷屏、欺騙 ...等相關行為，
違者 處以最低 <span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

# 3. 請勿破壞建築

包括 挖掘、放置、移動 ...等相關行為（也包括生物擊殺、變化），
此服規絕大多數時候僅保障 <a href="https://www.brilliantw.net/指令教學/#領地設置">有設置領地（含設好權限）</a>、<a href="https://www.brilliantw.net/指令教學/#鎖箱方法">有鎖箱</a> 或者 <a href="https://www.brilliantw.net/指令教學/#方塊查詢">具有 co 紀錄</a> 的玩家，
但這不意味著你可以毫無忌憚的破壞未設置領地、鎖箱的建築，最好就是不要破壞，那就不會有違規疑慮，
有關建築被破壞，欲申請補償者，請參考 <a href="hrilliantw.net/伺服守則/#其它-玩家建築物資被破壞、偷竊之補償標準">玩家建築被破壞之補償標準</a> 。
違者 處以最低 <span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

# 4. 請勿偷竊物資

包括 放入、拿取、惡意鎖牌、惡意鎖上物件 ...等相關行為，
此服規絕大多數時候僅保障 <a href="https://www.brilliantw.net/指令教學/#領地設置">有設置領地（含設好權限）</a>、<a href="https://www.brilliantw.net/指令教學/#鎖箱方法">有鎖箱</a> 或者 <a href="https://www.brilliantw.net/指令教學/#方塊查詢">具有 co 紀錄</a> 的玩家，
但這不意味著你可以毫無忌憚的竊取未設置領地、鎖箱的物資，最好就是不要偷，那就不會有違規疑慮，
有關物資被偷竊，欲申請補償者，請參考 <a href="hrilliantw.net/伺服守則/#其它-玩家建築物資被破壞、偷竊之補償標準">玩家物資被偷竊之補償標準</a> 。
違者 處以最低 <span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

# 5. 請勿擊打玩家

包括 打擊、惡意使其受到傷害 ...等相關行為（倘若對方同意，則不在此限），
違者 處以最低 <span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

# 6. 請勿利用漏洞

{% note warning %}
於 第三分流 - 很自由生電專用服 中，
利用「原版漏洞」並不屬在此規範範疇。
{% endnote %}

包括 複製、穿牆、宣傳漏洞（Exploit）...等相關行為，
違者 處以最低 <span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。

# 7. 請勿影響他人遊玩權益

包括 惡意設置領地、惡意競爭控價、商家廣告不實 ...等相關行為，
此規範之處分標準因較難很難明文規範其標準，故依輝煌管理人員決議為主，
倘若您對於欲進行的<span class="label label-info">遊戲行為</span> 有違規疑慮，請至 [Discord 群組](https://discord.brilliantw.net/) 詢問。

違者 處以最低 <span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。

如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。

## 7-1. 子規範 - 紅石

{% note warning %}
此規則不適用於 第三分流 - 很自由生電專用服。
{% endnote %}

在本服，我們禁止以下條件的<span class="label label-info">紅石機關</span> ：
{% note primary %}
一、作動速度 <span class="label label-warning">過快</span> 的機關。（例如：高頻連閃）

二、作動速度 <span class="label label-warning">過快</span> ，且會 <span class="label label-warning">自動運行</span> 的機關。（例如：高速環繞的漏斗礦車）
             （包括感測到訊號會運作的機關）

三、作動時，會 <span class="label label-info">推擠玩家</span> 的機關。（例如：有活塞推動的刷石機）
          
四、<span class="label label-info">一個區塊</span> 內，含有 <span class="label label-warning">5 座</span> 以上的紅石機關。

五、<span class="label label-info">一個區塊</span> 內，含有 <span class="label label-warning">100 個</span> 以上的 <span class="label label-info">漏斗</span> 。（包括漏斗礦車）
   倘若為建築裝飾用，無傳輸過任何物品，則不計入數量。
   使用 <span class="label label-info">/bhopper</span>指令 即可查看單區塊漏斗／漏斗礦車數量與限制。
          
但請注意！倘若因機關太大而影響伺服器，依然會處分！
{% endnote %}

只要您在本服建造<span class="label label-info">違反此規範類型之紅石機關</span> ，該<span class="label label-info">此紅石機關</span> 將被移除，
不會<span class="label label-warning">返還物資</span> ，
並處以最低 <span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。

有鑑於紅石機關的複雜性，我們很難明文規範其速率數值，
故對於該<span class="label label-info">紅石機關</span> 是否符合<span class="label label-success">合適速度</span> ，<span class="label label-warning">輝煌管理團隊</span> 擁有最終解釋權，
倘若您對於您建造的<span class="label label-info">紅石機關</span> 有違規疑慮，請至 [Discord 群組](https://discord.brilliantw.net/) 詢問。

## 7-2. 子規範 - 子帳戶

<span class="label label-info">主帳戶</span> 的定義：
{% note primary %}
在伺服器連線紀錄中，第一個以這個網路位置連入伺服器的玩家帳號。
{% endnote %}

<span class="label label-info">子帳戶</span> 的定義 包括但不限於：
{% note primary %}
一、不同 <span class="label label-warning">IP</span> ，但 <span class="label label-info">使用者相同</span> 的帳戶。

二、同 <span class="label label-warning">IP</span> ，且 <span class="label label-info">使用者相同</span> 的帳戶。

三、不同 <span class="label label-warning">平台</span> ，但 <span class="label label-info">使用者相同</span> 的帳戶。

四、同一 <span class="label label-warning">平台</span> ，且 <span class="label label-info">使用者相同</span> 的帳戶。

五、不同 <span class="label label-warning">分流</span> ，但 <span class="label label-info">使用者相同</span> 的帳戶。

六、同一 <span class="label label-warning">分流</span> ，且 <span class="label label-info">使用者相同</span> 的帳戶。
{% endnote %}

本服允許玩家同時登入 <span class="label label-warning">三個</span> 符合以上標準的 <span class="label label-info">子帳戶</span> ，
但當您同時登入的<span class="label label-info">子帳戶</span> 數量達到 <span class="label label-danger">第四個</span> 時，
相連的全部<span class="label label-danger">子帳戶</span> 將被停權，且<span class="label label-warning">主帳戶</span> 將被警告至少一次，
倘若您需要使用<span class="label label-info">第四個子帳戶</span> ，請至 [子帳申請](https://www.brilliantw.net/子帳申請) 了解詳細規範及申請流程。
簡單來說，一般情況下，您可以使用<span class="label label-warning">1</span>+<span class="label label-info">3</span>個帳戶同時遊玩輝煌伺服器。
> ---

註：<span class="label label-success">沒有同時登入</span> 就不會歸納在<span class="label label-info">子帳戶</span> 的範疇，
也就說如果你有<span class="label label-success">帳戶A</span> 跟<span class="label label-danger">帳戶B</span> ，
在<span class="label label-success">1時間段</span> 登入了<span class="label label-success">帳戶A</span> ，
而<span class="label label-danger">2時間段</span> 登入了<span class="label label-danger">帳戶B</span> ，
且<span class="label label-success">1時間段</span> 與<span class="label label-danger">2時間段</span> 不重疊到的話，
那就不會被判定為<span class="label label-info">子帳戶</span> 。

![子帳號機制簡單圖解](/img/伺服守則/sub.png)
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

## 7-3. 子規範 - 頻道使用

<span class="label label-info">聊天頻道</span> 中請勿進行任何 <span class="label label-warning">宣傳類型</span> 的交談，
可允許適度的 <span class="label label-info">討論交易內容</span> ，但建議使用 <span class="label label-success">/msg</span> 私訊討論，
包括 請求物資、要求收購、要求購買 ...等，
違者 處以最低 <span class="label label-warning">口頭警告</span> 最高 <span class="label label-warning">警告兩次</span> 之處分，

> ---

<span class="label label-info">交易頻道</span> 中，目前僅能進行以下項目的發言：
{% note primary %}
一、出售物品。 ( 任何交易形式 )

二、收購物品。 ( 任何交易形式 )

三、公共設施。 ( 任何交易形式 )

四、公會招募。 ( 邀請連結、文字招募、社群群組 )

五、其它任何交易形式發言
{% endnote %}

違者 處以最低 <span class="label label-warning">口頭警告</span> 最高 <span class="label label-warning">警告二次</span> 之處分。
頻道轉換、讀取 都可透過指令調整，相關指令請至 [指令教學網站](https://www.brilliantw.net/指令教學#頻道指令) 查看。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。

## 7-4. 子規範 - 卡服相關
有鑑於 Minecraft 本身的限制，即便伺服器主機處理器有著複數核心，但通常只能使用到最多一個核心的效能，
而隨者玩家的發展，逐漸有一些較為卡服的機器影響到玩家的遊戲體驗，固訂立此子規範。
{% note primary %}
一、有關於紅石相關規範，請至 <span class="label label-info">7-1. 子規範 - 紅石</span> 查看。

二、有關生物實體上限，請至 <span class="label label-info"><a href="https://www.brilliantw.net/指令教學/#實體">實體限制表格</a></span> 查看。

三、<span class="label label-warning">單一生怪塔</span> 的生產效率超過 <span class="label label-warning">50 隻 / 同時</span> 。
（也就是這個生怪塔不能同時存在 <span class="label label-danger">50 隻以上</span> 由 <span class="label label-info">生怪塔產出的生物</span> ）
（Java版 F3 有提供這個資訊，可自行判斷）
          
但請注意！如果生怪塔在未達標的情況下，影響了伺服器，依然會處分！
{% endnote %}

只要您在本服建造 <span class="label label-info">違反此規範類型之生怪塔</span> ，該 <span class="label label-info">生怪塔</span> 將被移除，
不會<span class="label label-warning">返還物資</span> ，
並處以最低 <span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。

對於該 <span class="label label-info">生怪塔</span> 是否符合 <span class="label label-success">規範效率與數量</span> ， <span class="label label-warning">輝煌管理團隊</span> 擁有最終解釋權，
倘若您對於您建造的 <span class="label label-info">生怪塔</span> 有違規疑慮，請至 [Discord 群組 💼・問題求助](https://discord.brilliantw.net/) 詢問。

{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。

## 7-6. 機率性遊戲設施建造規範
為避免機率性遊戲設施之廣告機率與獎勵不實、未公開機率等疑慮，
訂立此規範要求玩家以維持交易公平性，還請玩家務必遵守。
{% note primary %}
一、<span class="label label-info">機率性遊戲設施</span> 包括 抽獎機 .. 等帶有機率性獲利的實體設施。

二、需明確標示其 <span class="label label-info">獲獎機率與獎品</span> ，並且兩者需有聯繫。（例如 A大獎機率為 5%）

三、需明確標示其 <span class="label label-info">銷售價格</span> 。（例如 抽一次 100 乙太）

四、需明確標示其 <span class="label label-warning">操作方式</span> ，避免機台操作不當導致的損壞。

五、需明確標示其 <span class="label label-warning">出獎位置與投幣位置</span> ，避免機台操作不當導致的損壞。

六、有關出獎與 <span class="label label-info">機率調整之紅石設施</span>，需使用 <span class="label label-warning">發射器或者投擲器</span> 來建造。

七、業主提供給 <span class="label label-info">玩家之抽獎位置</span> ，需在出獎與機率調整之紅石設施 <span class="label label-warning">10 格位置以內</span> ，供其他玩家使用 <span class="label label-info">/cview</span> 查看。

八、業主提供給 <span class="label label-info">玩家之抽獎位置</span> ，需有 <span class="label label-warning">個人空間保障</span> ，當有玩家抽獎時。無法任意進出，避免物資盜竊之情事。

九、不可違反 <span class="label label-warning">中華民國刑法第266條~270條規定</span> 之內容。
{% endnote %}

玩家可於遊戲內使用 <span class="label label-info">/phome 服規7-6範例機台（一分流）</span> 指令，查看符合此規範類型的 <span class="label label-info">機率性遊戲設施</span> 。

只要您在本服建造 <span class="label label-info">違反此規範類型之機率性遊戲設施</span> ，該 <span class="label label-info">機率性遊戲設施</span> 將被移除，
不會<span class="label label-warning">返還物資</span> ，
並處以最低 <span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。

# 其它. 處分相關程度由 輕 至 重 如下
{% note primary %}
一、<span class="label label-info">口頭警告</span>

二、<span class="label label-warning">警告一次</span>

三、<span class="label label-warning">警告二次</span>

四、<span class="label label-warning">警告三次</span>

五、<span class="label label-danger">停權</span>
{% endnote %}

也請注意！處分並不是循序漸近，而是根據情況執行其一程度的處分！
而當警告累積達到 <span class="label label-warning">三次</span> 時，將予以 <span class="label label-danger">停權</span> 處分


# 其它. 玩家建築物資被破壞、偷竊之補償標準
為避免大量因玩家個人未做好保護而導致的行政負擔，
我們要求玩家要先做好個人資產的保護後，倘若依然有建築物資被破壞、偷竊之情事，
請依照下述標準申請補償：
{% note primary %}
一、在 <span class="label label-info">Discord 群組 💼・問題求助</span> 申請補償。

# 其它. 被封禁者之領地移除標準
{% note primary %}
一、在 <span class="label label-info">Discord 群組 💼・問題求助 申請</span> ，並提供 <span class="label label-info">領地名</span> 、 <span class="label label-info">被封禁者的遊戲名稱</span> 。

二、該領地的 <span class="label label-success">擁有者</span> 必須為 <span class="label label-danger">被封禁者</span> 。

三、申請移除者 必須是該領地的 <span class="label label-success">擁有者</span>  或者 <span class="label label-secondary">玩家權限列表成員之一</span> 。

四、倘若該領地的 <span class="label label-secondary">玩家權限列表成員</span> 僅有 被<span class="label label-danger">封禁者（或者沒有）</span>，則可無視 <span class="label label-info">第五點</span> 標準直接移除。

五、倘若該領地的 <span class="label label-secondary">玩家權限列表成員</span> 中，有 <span class="label label-info">其他玩家</span> 之權限，需先獲得 <span class="label label-info">其他玩家</span> 的同意後（提供對話截圖），才可移除。

六、移除 被封禁者之領地 <span class="label label-success">須完全達成</span> 第 <span class="label label-info">一、二、三、四</span> ／ <span class="label label-info">一、二、三、五</span> 項標準，缺一不可。

七、被封禁者 的領地將於 封禁 <span class="label label-warning">90 天後</span>，由系統自動移除，不需申請。
{% endnote %}
