# 1. 請勿使用外掛

包括 飛行、RNG計算器 ...等，
修改遊戲數據或嚴重影響遊戲公平性之行為。
違者<span class="label label-danger">必定停權</span> 。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

# 2. 請勿冒犯他人

包括 威脅、淫穢、誹謗、仇恨、騷擾、侮辱、刷屏、欺騙 ...等相關行為，
違者 處以最低<span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

# 3. 請勿破壞建築

包括 挖掘、放置、移動 ...等相關行為（也包括生物擊殺、變化），
違者 處以最低<span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

# 4. 請勿偷竊物資

包括 放入、拿取、惡意鎖牌、惡意鎖上物件 ...等相關行為，
違者 處以最低<span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

# 5. 請勿擊打玩家

包括 打擊、惡意使其受到傷害 ...等相關行為（倘若對方同意，則不在此限），
違者 處以最低<span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

# 6. 請勿利用漏洞

包括 複製、穿牆、宣傳漏洞（Exploit）...等相關行為，
違者 處以最低<span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

# 7. 請勿影響他人遊玩權益

包括 大型機器、大量生物、使用分帳號 ...等相關行為，
違者 處以最低<span class="label label-warning">口頭警告</span> 最高<span class="label label-danger">停權</span> 之處分。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

## 7-1. 子規範 - 紅石

在本服，我們禁止以下條件的<span class="label label-info">紅石機關</span> ：
{% note danger %}
1. 作動速度 <span class="label label-warning">過快</span> 的機關。（例如：高頻連閃）

2. 作動速度 <span class="label label-warning">過快</span> ，且會 <span class="label label-warning">自動運行</span> 的機關。（例如：高速環繞的漏斗礦車）
             （包括感測到訊號會運作的機關）

3. 作動時，會 <span class="label label-info">推擠玩家</span> 的機關。（例如：有活塞推動的刷石機）
          
4. <span class="label label-info">一個區塊</span> 塊內，含有 <span class="label label-warning">5 座</span> 以上的紅石機關。

5. <span class="label label-info">一個區塊</span> 內，含有 <span class="label label-warning">50 個</span> 以上的 <span class="label label-info">漏斗</span> 。（包括漏斗礦車）
   倘若為建築裝飾用，無須傳輸物品或移動使用，則不計入數量。
   計數標準：
   - <span class="label label-info">一個漏斗</span> 正在傳輸物品，計為 <span class="label label-warning">一個漏斗</span> 。
   - <span class="label label-info">一個漏斗</span> 傳輸物品的頻率低於 <span class="label label-danger">30 秒</span> ，計為 <span class="label label-warning">一個漏斗</span> 。
     （例如在第 1 秒傳輸了 一個石頭，在第 10 秒又傳輸了 一個石頭）
   - <span class="label label-info">一個漏斗礦車</span> 正在傳輸物品，計為 <span class="label label-warning">一個漏斗礦車</span> 。
   - <span class="label label-info">一個漏斗礦車</span> 傳輸物品的頻率低於 <span class="label label-danger">30 秒</span> ，計為 <span class="label label-warning">一個漏斗礦車</span> 。
     （例如在第 1 秒傳輸了 一個石頭，在第 10 秒又傳輸了 一個石頭）
   - <span class="label label-info">一個漏斗礦車</span> 正在移動，，計為 <span class="label label-warning">一個漏斗礦車</span> 。
          
但請注意！倘若因機關太大而影響伺服器，依然會處分！
{% endnote %}

只要您在本服建造<span class="label label-info">此規範類型之機關</span> ，該<span class="label label-info">此規範類型之機關</span> 將被移除，
且<span class="label label-warning">主帳戶</span> 將被至少警告一次，並且不會<span class="label label-warning">返還物資</span> ，
有鑑於紅石機關的複雜性，我們很難明文規範其速率數值，
故對於該<span class="label label-info">紅石機關</span> 是否符合<span class="label label-success">合適速度</span> ，<span class="label label-warning">輝煌管理團隊</span> 擁有最終解釋權，
倘若您對於您建造的<span class="label label-info">紅石機關</span> 有違規疑慮，請至 [Discord 群組](https://discord.com/invite/5MHGpAFGEN) 詢問。

## 7-2. 子規範 - 子帳戶

<span class="label label-info">子帳戶</span> 的定義 包括但不限於：
{% note danger %}
1. 同 <span class="label label-warning">IP</span> 帳戶。

2. 不同 <span class="label label-warning">平台</span> ，但 <span class="label label-info">使用者相同</span> 的帳戶。

3. 不同 <span class="label label-warning">IP</span> ，但 <span class="label label-info">使用者相同</span> 的帳戶。

4. 同一 <span class="label label-warning">平台</span> ，且 <span class="label label-info">使用者相同</span> 的帳戶。
{% endnote %}

只要您使用了未經申請通過的<span class="label label-danger">子帳戶</span> 。
該<span class="label label-danger">子帳戶</span> 將被停權，且<span class="label label-warning">主帳戶</span> 將被警告至少一次，
倘若您需要使用<span class="label label-info">子帳戶</span> ，請至 [子帳申請](https://www.brilliantw.net/子帳申請) 了解詳細規範及申請流程。

> ---

註：<span class="label label-success">沒有同時登入</span> 就不會歸納在<span class="label label-info">子帳戶</span> 的範疇，
也就說如果你有<span class="label label-success">帳戶A</span> 跟<span class="label label-danger">帳戶B</span> ，
在<span class="label label-success">1時間段</span> 登入了<span class="label label-success">帳戶A</span> ，
而<span class="label label-danger">2時間段</span> 登入了<span class="label label-danger">帳戶B</span> ，
且<span class="label label-success">1時間段</span> 與<span class="label label-danger">2時間段</span> 不重疊到的話，
那就不會被判定為<span class="label label-info">子帳戶</span> 。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

## 7-3. 子規範 - 頻道使用

<span class="label label-info">聊天頻道</span> 中請勿進行任何<span class="label label-warning">宣傳類型</span> 的交談，
可允許適度的<span class="label label-info">討論交易內容</span> ，但建議使用<span class="label label-success">/msg</span> 私訊討論，
包括 請求物資、要求收購、要求購買 ...等，
違者 處以最低<span class="label label-warning">口頭警告</span> 最高<span class="label label-warning">警告兩次</span> 之處分，

> ---

<span class="label label-info">貿易頻道</span> 中，目前僅能進行以下項目的發言：
{% note success %}
1. 出售物品。 ( 任何交易形式 )

2. 收購物品。 ( 任何交易形式 )

3. 公共設施。 ( 任何交易形式 )

4. 公會招募。 ( 邀請連結、文字招募、社群群組 )

5. 其它任何交易形式發言
{% endnote %}

違者 處以最低<span class="label label-warning">口頭警告</span> 最高<span class="label label-warning">警告二次</span> 之處分。
頻道轉換、讀取 都可透過指令調整，相關指令請至 [指令教學網站](https://www.brilliantw.net/指令教學#頻道指令) 查看。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

## 7-4. 子規範 - 無領地權益

倘若你的建築沒有設置領地，遭受到了破壞、偷竊，
想透過服規 3、4 來檢舉該名玩家，你需要做到以下幾點 ＂ 之一 ＂，這項檢舉才會生效：
{% note warning %}
1. 事先在建築 <span class="label label-info">「顯眼」</span> 的地方放上任何能明確代表此建築是屬於您的物件。（例如放告示牌標明請勿破壞）
             └ 顯眼的意思指的是一眼就看得到，而不是放在小角落或者隱密處。
             
2. 在 <span class="label label-danger">破壞、偷竊者</span> 在線時給予 <span class="label label-warning">口頭提醒</span> ，而 <span class="label label-danger">破壞、偷竊者</span> 在 <span class="label label-warning">口頭提醒</span> 後再犯。
{% endnote %}

但也還請注意！這不是你可以不設置領地的理由，
這項規則只適用一次，也就是你不能第二次使用這個規則，
當你想要第二次使用這個規則來檢舉他人時，我們將不會受理，
即便你達成了以上的條件，還請留意。
{% note danger %}
如有未被此規範舉例，但屬此規範範圍之例外情況，依輝煌管理人員決議為主。
{% endnote %}

## 7-5. 子規範 - 卡服相關
{% note warning %}
此規範將於 2023年 12月 09日 14時 30分 正式啟用，建議玩家在此之前進行相關改動。
{% endnote %}
有鑑於 Minecraft 本身的限制，即便伺服器主機處理器有著複數核心，但通常只能使用到最多一個核心的效能，
而大部分伺服器選擇加開分流來應對這個限制，而本服目前沒有加開分流的打算，
但隨者玩家的發展，逐漸有一些較為卡服的機器影響到玩家的遊戲體驗，固訂立此子規範。
{% note danger %}
1. 有關於紅石相關規範，請至 <span class="label label-info">7-1. 子規範 - 紅石</span> 查看。
          
2. 在 <span class="label label-warning">10 × 10 個區塊</span> 內，含有 <span class="label label-warning">1 個</span> 之上的生怪塔。
（不含 <span class="label label-danger">1 個</span> ，生怪塔需在 <span class="label label-danger">10 × 10 區塊</span> 的 <span class="label label-info">中心位置</span>）

3. <span class="label label-warning">單一生怪塔</span> 的生產效率超過 <span class="label label-warning">50 隻 / 同時</span> 。
（也就是這個生怪塔不能同時存在 <span class="label label-danger">50 隻以上</span> 由 <span class="label label-info">生怪塔產出的生物</span> ）
（Java版 F3 有提供這個資訊，可自行判斷）
          
但請注意！如果生怪塔在未達標的情況下，影響了伺服器，依然會處分！
{% endnote %}

只要您在本服建造<span class="label label-info">此規範類型之生怪塔</span> ，該<span class="label label-info">此規範類型之生怪塔</span> 將被移除，
且<span class="label label-warning">主帳戶</span> 將被至少警告一次，並且不會<span class="label label-warning">返還物資</span> ，
故對於該<span class="label label-info">生怪塔</span> 是否符合<span class="label label-success">規範效率與數量</span> ，<span class="label label-warning">輝煌管理團隊</span> 擁有最終解釋權，
倘若您對於您建造的<span class="label label-info">生怪塔</span> 有違規疑慮，請至 [Discord 群組](https://discord.com/invite/5MHGpAFGEN) 詢問。

# 其它. 處分相關程度由 輕 至 重 如下

          1. 口頭警告

          2. 警告一次

          3. 警告二次

          4. 警告三次

          5. 停權  

也請注意！處分並不是循序漸近，而是根據情況執行其一程度的處分！

# 其它. 發放全服補償禮包之標準

          一、單次維護時長超過 12小時（含） 以上。
          
          二、玩家資料遺失等重大伺服器損壞情況。
          
          三、單次維護時間跨日期。（如 7/7 維護到 7/8 ）

# 其它. 玩家建築被破壞、偷竊之補償標準

          一、在 Discord 群組 事務頻道 申請補償。
          
          二、申請補償時，明確提供被破壞、偷竊位置的 方塊紀錄 之 圖片、影片 ...等 媒體證明。
          
          三、有設置領地，並且領地權限有設置完全。
          
          四、在被破壞、偷竊後的近 30 天 內申請補償。（超過 30 天會沒有方塊紀錄）
          
          五、申請補償時，需明確提供 建築所在分流、位置、建造者的遊戲名稱 與 破壞 / 偷竊者的遊戲名稱。

# 其它. 被封禁者之領地移除標準

          一、在 Discord 群組 事務頻道 申請，並提供領地名、被封禁者的遊戲名稱。

          二、該領地的 擁有者 必須為 被封禁者。

          三、申請移除者 必須是該領地的 擁有者 或者 玩家權限列表成員之一。

          四、倘若該領地的 玩家權限列表成員 僅有 被封禁者（或者沒有），則可無視 第五點 標準直接移除。

          五、倘若該領地的 玩家權限列表成員 中，有 其他玩家 之權限，需先獲得 其他玩家 的同意後（提供對話截圖），才可移除。

          六、移除 被封禁者之領地 須完全達成 第 一、二、三、四 ／ 一、二、三、五 項標準，缺一不可。

          七、被封禁者 的領地將於 封禁 30 天後，由系統自動移除，不需申請。
